# Freelancer Memory Walkthrough

Freelancer Memory helps you reply faster without forgetting the important stuff.

Pricing.
Scope.
Payment terms.
Deadlines.
Boundaries.
Client details.

Simple.

## Start Here

1. Open the extension.
2. Go to `My memory`.
3. Add your freelance basics once:
   - role
   - services
   - pricing
   - proof
   - payment terms
   - boundaries
   - voice
4. Open a client thread in Gmail, LinkedIn, Upwork, Fiverr, WhatsApp Web, X, or any website.
5. Click `Use page`.
6. Click `Generate reply`.
7. Review the reply.
8. Accept useful memory updates.
9. Click `Insert` or `Copy`.

That is the main workflow.

## What To Add In My Memory

Keep it short.

Example:

```txt
Role: Webflow developer
Services: SaaS landing pages and Webflow builds
Pricing: Landing pages start at $2,000
Payment terms: 50% upfront, 50% before launch
Proof: Built landing pages for 12 SaaS teams
Boundaries: No unlimited revisions. No unpaid strategy calls.
Voice: Direct, warm, short
```

Set this once.
Update it when replies feel wrong.

## Daily Use

Use Freelancer Memory before replying to a client.

Good client messages:

```txt
How much would this cost?
Can you finish by Friday?
Can we add one more page?
Can we skip the upfront payment?
What is included?
```

The app uses your saved memory plus the client message to draft a better reply.

Example:

```txt
Saved memory:
Landing pages start at $2,000.
50% upfront.
Dashboard pages are not included.

Client:
Can you also add dashboard pages?

Reply:
That is extra scope. I can add it as a paid add-on, or we can swap it with something already included.
```

This is the point:

Stop guessing.
Stop repeating yourself.
Stop saying yes to unpaid scope.

## Use Page vs Use Highlight

Use `Use page` first.

It reads the current client thread.

Use `Use highlight` when the page is messy or the app grabs the wrong text.

Do this:

1. Highlight only the client message.
2. Click `Use highlight`.
3. Check it appears in `Client message`.
4. Click `Generate reply`.

Good highlight:

```txt
Can you add a pricing page too? We still need launch by Friday.
```

Bad highlight:

```txt
The whole page, nav, footer, sidebar, ads, and random text.
```

Small context wins.

Important:

```txt
Highlight does not save text to My memory.
```

It only uses the selected text for the current reply.

## Client And Project Memory

Client memory remembers who the client is.

Project memory remembers what was agreed.

Project memory matters most.

Save facts like:

- budget
- timeline
- included scope
- excluded scope
- payment terms
- risks
- next step

Example:

```txt
Project: Landing page redesign
Budget: $2,500
Timeline: 10 days
Included: homepage redesign, responsive Webflow build
Excluded: dashboard pages, full copy rewrite
Payment terms: 50% upfront, 50% before launch
Next step: Send final scope for approval
```

Now when the client asks for extra work, the app can catch it.

That protects your money.

## Memory Review

After generating a reply, the app may suggest memory updates.

Review anything that affects:

- money
- time
- scope
- payment
- risk

Accept what is true.
Reject what is wrong.

Blunt rule:

```txt
If it changes the deal, review it before saving.
```

## Buttons

`Use page`

Reads the current page.

`Use highlight`

Uses only selected text.

`Generate reply`

Creates a reply with your memory.

`Insert`

Adds the reply to the reply box.

`Copy`

Copies the reply if insert fails.

`Active client`

Pick the right client.

`Active project`

Pick the right project.

`Accept all`

Save all suggested memory updates. Use carefully.

`Reject all`

Ignore all suggested updates.

## Common Fixes

### The wrong client is showing

Use `Active client` and select the right one.

### Project memory is empty

Select a project or save/import the project details first.

### Insert does not work

Click inside the reply box, then try `Insert` again.

If it still fails, use `Copy`.

### Highlight did not save to My memory

That is normal.

Highlight is only for the current message.
Edit `My memory` manually.

## Best Habit

Before every important client reply:

1. Select the right client/project.
2. Use page or highlight the message.
3. Generate the reply.
4. Check for pricing, scope, deadline, and payment issues.
5. Accept only true memory updates.
6. Send.

Fast replies.
Fewer mistakes.
Better boundaries.
